# ForgeStack

**ForgeStack** is a command-line tool designed to rapidly scaffold modern React applications with a predefined architecture and tooling stack. It eliminates repetitive project setup tasks and allows developers to start working directly on application logic instead of infrastructure.

---

## Overview

Starting a new frontend project typically involves repeating the same setup steps:

- Creating a Vite project
- Configuring Tailwind CSS
- Setting up Shadcn UI
- Adding React Router
- Creating an Axios client
- Setting up React Query and Zustand
- Building form components with React Hook Form and Zod
- Implementing authentication guards
- Creating layouts and utilities

ForgeStack automates these steps by generating a fully configured project using a predefined template.

With a single command, developers can scaffold a production-ready project structure that already includes all the necessary tools and conventions.

---

## Key Goals

ForgeStack is designed with the following objectives:

### Reduce Boilerplate

Developers should not spend hours configuring the same tools every time a new project is created.

### Enforce a Consistent Architecture

Every project generated with ForgeStack follows the same directory structure, naming conventions, and tooling setup.

### Improve Developer Experience

The CLI guides the user through project creation with simple prompts and automatically performs the setup.

### Accelerate Project Kickoff

Instead of spending time configuring tools, developers can immediately begin implementing business logic.

## How ForgeStack Works

ForgeStack operates as a CLI scaffolding engine.

When executed, the CLI:

1. Prompts the user for basic configuration values
2. Downloads the template repository
3. Injects user-defined configuration into the template
4. Removes template-specific metadata
5. Installs project dependencies
6. Outputs a ready-to-run project

Example workflow:

```bash
npx forge-stack
```

The CLI prompts the user:

```
Project name: my-app
API Base URL: https://localhost:8443/api/v1
```

ForgeStack then generates the project and installs dependencies.

The developer can immediately start the project:

```bash
cd my-app
npm run dev
```

## Generated Project Architecture

ForgeStack generates projects using a hybrid architecture combining global components with feature-based organization.

Example structure:

```
src
 ├── app
 │   ├── App.tsx
 │   ├── providers.tsx
 │   └── router.tsx
 │
 ├── components
 │   ├── ui
 │   ├── forms
 │   ├── layouts
 │   └── common
 │
 ├── features
 │   ├── auth
 │   ├── dashboard
 │   └── users
 │
 ├── hooks
 ├── lib
 ├── utils
 ├── store
 ├── enums
 └── constants
```

This architecture separates infrastructure, shared UI components, and domain-specific features.

## Configuration Injection

ForgeStack allows injecting configuration values during project generation.

Currently supported:

- **Project name**
- **API base URL**

These values are automatically applied to:

- `package.json`
- `.env`
- template placeholders

## Roadmap

ForgeStack is designed to evolve into a more powerful scaffolding engine.

Planned features include:

### Feature Selection

Allow developers to enable optional modules during project creation:

- Authentication
- React Query
- Zustand
- Dark mode
- Internationalization

### Theme Injection

Support applying **TweakCN themes** during project generation.

### Project Profiles

Provide predefined profiles such as:

- Dashboard
- SaaS application
- Admin panel
- Landing page

### Modular Architecture

Allow ForgeStack to dynamically assemble projects using modular template components instead of a single static template.

### Plugin System

Enable external packages to extend ForgeStack with additional modules.

## Installation

ForgeStack is distributed via npm and can be executed directly using `npx`.

```bash
npx forge-stack
```

No global installation is required.

---

## Philosophy

ForgeStack is built around a simple idea:

By automating repetitive setup tasks and providing a standardized architecture, ForgeStack aims to significantly accelerate the development of modern frontend applications.
